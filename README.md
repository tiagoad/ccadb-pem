# ccadb-pem

This repository contains code that fetches certificates from the [CCADB](https://www.ccadb.org/) (Common CA Database)
and outputs them as .pem files containing several different subsets of the database certificates.

## Why I built this

While working on scraping tasks, I sometimes come across websites that don't send the full certificate chain
in HTTPS responses. This is fine when accessing the website through a browser, but it will fail when making the
same requests through Node.js, for example.

In node, this is fixed by setting the `NODE_EXTRA_CA_CERTS` environment variable to the path to a .pem file
containing intermediate CA certificates, like the ones generated by this tool.

## How to use

.pem files are generated daily by a Github Action and made available at https://tiagoad.github.io/ccadb-pem/

### Generating the files yourself

Node.js version 26 or higher is required, along with [pnpm](https://pnpm.io/installation).

1. Install dependencies  
   `pnpm install`

2. Generate the pem files  
   `pnpm run generate`

You should end up with a bunch of `.pem` files in the `output` directory.

## Available files

The first part of the filename determines who trusts a given certificate:

- `apple`: Trusted by Apple
- `chrome`: Trusted by Google Chrome
- `microsoft`: Trusted by Microsoft
- `mozilla`: Trusted by Mozilla
- `all`: Trusted by all of the above
- `any`: Trusted by at least one of the above
- `notrust`: All certificates in the DB are included, including ones not trusted by the above entities (excluding revoked certificates)

The second part of the filename determines the types of certificates included:

- `root`: Root Certificates only
- `intermediate`: Intermediate Certificates only
- `both`: Both types
